# 東洋医学 学習スイート

国試学習・経穴検索・弁証支援・経穴暗記の4アプリを、1つのホーム画面から起動できるようにまとめた統合 PWA です。ホームに4つのカードを表示し、タップすると各アプリが全画面で開きます（iframe 方式）。

## 特長

- **4アプリを1か所に** — ホームの4カードから各アプリへ分岐。「‹ ホーム」で戻れます。
- **既存データはそのまま** — 各アプリの学習履歴・カルテは従来どおり端末内（localStorage / IndexedDB）に保存され、これまでのデータを引き継いで使えます。
- **バージョンは1つに統一** — `version.json` で一元管理。ホーム右下の「🔄 更新確認」だけで全アプリまとめて最新化されます。
- **日付バージョン＋自動化** — push すると GitHub Actions が日付でバージョンを自動生成します（例 `2026.6.4`。同じ日に複数回更新したときだけ末尾に短いハッシュが付きます）。
- **オフライン対応** — 一度開けば通信なしでも起動・利用できます。

```
├── version.json            バージョン（GitHub Actions が日付で自動更新）
├── .nojekyll               GitHub Pages 用（必須・中身は空）
├── icons/
│   ├── icon-192.png
│   ├── icon-512.png
│   └── icon-512-maskable.png
├── apps/
│   ├── kokushi/            国試学習アプリ
│   │   ├── index.html
│   │   ├── manual.html
│   │   └── questions.json  （問題データ。ここを編集すれば問題を更新できます）
│   ├── search/             経穴検索アプリ
│   │   ├── index.html
│   │   └── acupoints.json  （経穴データ。ここを編集すれば内容を更新できます）

```

## 公開手順（GitHub Pages）

1. 上記ファイル一式をリポジトリのルートに配置して push。
2. **Settings → Actions → General → Workflow permissions** で「**Read and write permissions**」を有効化（version.json の自動コミットに必要）。
3. **Settings → Pages → Build and deployment → Source** で「**GitHub Actions**」を選択。
4. main（または master）に push すると、ワークフローが
   1. `version.json` を日付で更新（例 `2026.6.4`）
   2. GitHub Pages へデプロイ
   を自動実行します。
5. 数十秒〜数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

## 更新のしくみ

- 各アプリ本体（HTML）やデータ（`questions.json` / `acupoints.json`）を変更して push すると、`version.json` が日付で自動更新されます。
- 利用者がホームの「🔄 更新確認」を押すと、新しいバージョンがあれば Service Worker が全ファイルを取り直してキャッシュを入れ替え、最新版に切り替わります。
- 重いデータ（`questions.json` 約7.7MB、`acupoints.json`）は、そのアプリを初めて開いたときに取得・キャッシュされます。ホームの起動を軽く保つための遅延キャッシュ方式です。

### 個別アプリだけ直したいとき

- 問題を増やすだけ → `apps/kokushi/questions.json` を編集して push。
- 経穴データを直すだけ → `apps/search/acupoints.json` を編集して push。
- アプリの見た目・動作を直す → 該当する `apps/＜アプリ＞/index.html` を編集して push。

いずれも push すれば日付バージョンが自動で上がり、利用者側に更新が届きます。

## オンライン専用（著作権管理）

著作権管理のため、このアプリは**オンライン専用**です。コンテンツ（問題・経穴データ等）は端末に永続保存されず、オンライン時にサーバーから取得して、その場限りで表示します（タブを閉じると消えます）。

- 起動時・24時間無操作後の操作時・バックグラウンドから復帰した瞬間に、`version.json` をサーバーへ確認します。
- バージョンが変わっていれば、自動的に最新版へ強制更新します。
- 通信できない（オフライン等）ときは、コンテンツを表示せず「再読み込み」ボタン付きの画面を出します（真っ白画面になりません）。状態の一時情報（sessionStorage）も消去します。
- 利用者の**学習履歴・カルテは従来どおり端末内に保存**され、これとは別に保持されます。

> オフラインでは利用できません（電車内・電波の弱い場所など）。これは著作権管理を優先した仕様です。

### 制作者情報・バージョン

制作者情報（学校名・氏名・連絡先）とバージョンは、**統合ホームにのみ**表示されます。各アプリ個別の「更新」ボタン・バージョン表示は撤去しました。

## 仕組みのメモ（iframe 方式）

各アプリは元のコードをほぼそのまま `apps/＜アプリ＞/` に配置しています。統合のために、各 `index.html` の先頭へ短いガードスクリプトを1つだけ追加してあります。これは「iframe 内で動くときだけ、そのアプリ独自の Service Worker 登録・キャッシュ操作・更新ボタンを無効化する」ものです（単体で直接開いたときは元のまま動作します）。キャッシュとバージョン管理を親（このホーム）に一本化し、アプリ同士の干渉を防ぐためのものです。

## ローカル確認

`file://` で直接開くと Service Worker は無効ですが、ホームと各アプリの基本動作は確認できます。PWA 動作まで確認するには簡易サーバーを使ってください。

```
python3 -m http.server 8000
# → http://localhost:8000/ を開く
```

## 注意

- 公開リポジトリの場合、問題データ・経穴データは第三者からも閲覧可能です。共有先にご注意ください。
- iOS Safari は一定期間アプリを開かないと端末内データが削除される場合があります。各アプリのバックアップ機能（JSON 出力）を定期的にご利用ください。
